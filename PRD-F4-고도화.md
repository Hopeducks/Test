## 0. 한 줄 요약

학습 몰입도를 높이기 위해 **(1) 새어나가는 경제를 마일스톤 기반으로 재설계**하고, **(2) 죽어있는 업적·퀘스트·코스튬 시스템의 결함을 수정**하며, **(3) 아바타 상품과 도전과제를 대량 확장**하고, **(4) 구식 대기실 로비를 절차적 아트로 게임답게 고도화**한다.

### 1.2 핵심 인사이트 (학습 동기 설계)

> "너무 쉽게 포인트를 획득하면 학습 흥미가 쉽게 저하됨."

## 3. 기능 요구사항

### 3.1 [EPIC A] 경제 재설계 — "마일스톤만 지급" (R3, D1·D5·D6)

#### A-1 단일 코인 게이트웨이
- 모든 코인 증감을 `gameStateManager`의 **단일 메서드 `awardCoins(source, amount)` / `spendCoins(amount)`**로 일원화한다.
- `progress.coins`를 **유일한 진실 원천**으로 삼고, `coins ?? cards*10` fallback 산식을 전부 제거(D6). 마이그레이션 1회만 기존 사용자 보정.

#### A-2 직접 지급 제거 (퍼주기 금지)
다음 경로의 **직접 코인 지급을 0으로** 한다:
- 퀴즈 정답/완료(`UnitComplete.completeUnit`의 `score*10`)
- 카드 해금(`unlockCard`의 +30)
- 카드 배틀 승리(`CardBattleArena` +10~35)
- 포켓몬 센터(+10)
- 단, **전투 경험치(XP)→레벨 연동은 유지·강화**한다(R3 후단). 배틀/퀴즈는 **코인 대신 카드 XP·트레이너 XP**를 준다.

#### A-3 코인 획득은 마일스톤에서만
코인은 아래 **달성형(1회성) 이벤트**에서만 발생:
| 출처 | 조건 | 비고 |
|------|------|------|
| 첫 단원 클리어 보너스 | 해당 단원 최초 완료 1회 | `completedUnits`에 신규 추가될 때만 |
| 신기록 갱신 보너스 | `unitHighScores[unit]` 경신 시, 증가분 비례 | 재플레이 farming 차단 |
| 퀘스트 보상 | 티어 달성 후 수령 | 기존 `claimQuestReward` 유지 |
| 업적 보상 | 업적 최초 달성 | EPIC B에서 동작화 |
| 체육관 첫 격파 | 단원별 최초 1회 | `gymLeaderBeaten[unit]` 가드 |

#### A-4 검증 가능 기준 (AC)
- 같은 단원을 만점으로 10회 반복해도 코인 총량은 1회차(첫클리어+신기록)와 동일.
- 코인 증가는 전부 `awardCoins(source,...)` 호출을 거치며, source 로그로 추적 가능.
- 기존 저장 데이터 로드 시 음수/NaN 코인 없음.

### 3.2 [EPIC B] 업적·퀘스트 시스템 동작화 + 대량 확장 (R2, D2·D3·D4·D7)

#### B-1 업적 엔진 연결 (D2·D7)
- `checkAchievements(event)`를 **이벤트 발생 지점에 실제 연결**: 퀴즈 정답(streak), 단원 완료(unit_complete), 카드 해금(questions_correct), 레벨업(level), 보스 데미지(boss_damage), 배틀 승리(battles_won).
- 획득 업적을 `progress.earnedAchievementIds[]`로 **영속 저장**(D7). `Player.achievements`는 이 배열을 읽도록 수정.
- 업적 달성 시 토스트/모달 알림 + 보상 지급.

#### B-2 보상 타입 전체 처리 (D3)
- `reward.type` 분기에 `'title'`, `'card'` 추가. title 보상은 칭호 코스튬 해금으로 매핑, card 보상은 `unlockCard`.

#### B-3 코스튬 ID 정합성 (D4)
- `checkMilestones()`/achievements가 참조하는 **존재하지 않는 코스튬 ID를 카탈로그에 실제 추가**하거나, 카탈로그의 실재 ID로 교체. 단일 진실원천은 `costume-catalog.ts`.
- CI성 단위 테스트 추가: "모든 achievement.reward.costumeId와 checkMilestones 매핑은 카탈로그에 존재한다"를 단언(회귀 방지).

#### B-4 도전과제 대량 확장 (30 → 80+)
빠른 학생의 long-tail 몰입을 위해 카테고리·티어 확장:
- 기존 6개 퀘스트 카테고리 유지 + 신규 카테고리 추가:
  - **속도/숙련**: 제한시간 내 정답, 노힌트 단원 클리어, 단원 만점(퍼펙트)
  - **완벽주의**: 오답노트 0개 유지, 연속 단원 만점
  - **수집 심화**: 등급별 컬렉션(레전더리 8종, epic 8종 전부)
  - **육성**: 특정 카드 레벨 10 도달, 진화 3단계 달성 카드 N개
  - **배틀/레이드**: 누적 승수, 보스 막타, 무피해 승리
- 각 카테고리 5티어 → 도전과제 총 **80개 이상**. 상위 티어는 "도감 완성/전 단원 퍼펙트" 등 초고난도로 빠른 학생도 수개월 분량 확보.
- **일일/세션 도전과제**(선택): 날짜 시드 기반 3종 회전(반복 가능, 단 코인 상한은 A-3 정책 내).

#### B-5 AC
- 신규 세션에서 streak 3회 달성 → 해당 업적이 즉시 earned 처리되고 보상 지급, 새로고침 후에도 유지.
- 카탈로그 정합성 테스트 통과(미존재 ID 0건).
- 퀘스트 패널에 80+ 도전과제가 카테고리별로 노출되고 진행도 정확.

### 3.3 [EPIC C] 아바타 상품 다양화 (R1, D9)

#### C-1 카탈로그 확장 (46 → 90+)
- 카테고리별 신규 상품(이모지 레이어로 표현 가능한 것 위주):
  - 의상 +8, 모자 +6, 악세서리 +8, 탈것 +4, 펫 +6, 칭호 +6, **신규: 표정/이펙트(파티클 색·오라) +6**
- **등급 4단계화**: common/rare/epic/legendary로 확장(D9 해소: `CostumeItem.rarity`에 epic 추가, epic 카드와 정합). 등급별 시각적 변별(테두리·광원·파티클).
- 가격·해금조건 밸런싱: A-3 경제와 정합(고가 상품은 마일스톤/고티어 퀘스트 보상).

#### C-2 변별력·세트 효과
- **세트 보너스**: 같은 테마(예: 우주복+우주헬멧+로켓) 전부 장착 시 CP 보너스 + 전용 칭호 노출.
- 코스튬 stats가 CP에 반영되는 현 구조 유지하되 EPIC D 통합 산식에 편입.

#### C-3 AC
- 카탈로그 90종 이상, 전 항목이 미리보기(`AvatarPreviewPanel`)·로비(`LobbyScene.updateAvatar`)에서 정상 렌더.
- 미해금/해금/장착 상태 전이가 일관(중복 컴포넌트 `components/AvatarCustomizer` vs `components/ui/avatar` 정리 포함).

### 3.4 [EPIC D] 포켓몬 월드 육성 통합 (R4, D8)

#### D-1 단일 진척 모델
- 흩어진 산식(`getTrainerInfo` XP, `getLocalPlayer().level`, `calculateCP`)을 **하나의 `deriveProgression(progress)` 순수함수**로 통합. 트레이너 레벨/XP/CP/카드레벨 합을 한 곳에서 파생.
- "로그인 학생"의 캐릭터 육성이 퀴즈→카드XP→카드레벨/진화→트레이너 랭크→CP→배틀력으로 **일관 연결**되도록 정의.

#### D-2 전투 경험치 = 레벨 (R3 후단)
- 퀴즈 정답·배틀·레이드는 코인 대신 **카드 XP / 트레이너 XP**를 지급(이미 부분 구현된 `gainCardXp` 강화).
- 카드 진화(`getCardEvolution`)와 레벨 보상을 UI에 명확히 노출(레벨업 토스트는 존재 → 진화 연출 보강).

#### D-3 AC
- 트레이너 레벨·CP가 모든 화면(로비 사이드바, 마이페이지, 배틀)에서 동일 값.
- 배틀 승패가 카드 XP에 반영되고, 카드 레벨↑ → CP↑ → 표시 일관.

### 3.5 [EPIC E] 대기실 로비 게임화 — 절차적 아트 고도화 (R5)

> 외부 에셋 0. `LobbyScene.ts`의 Canvas/Graphics 절차 생성을 고도화한다.

#### E-1 배경·환경 아트
- 단조로운 32px 격자 → **존별 테마 바닥**(퀴즈존=단원 색 그라데이션, 배틀=용암 텍스처, 레이드=보라 성운, 박물관=대리석, 센터=핑크 타일, 체육관=금속). Canvas 멀티 타일 + 노이즈/디더링으로 질감.
- **장식 오브젝트**: 나무·분수·표지판·홀로그램 기둥·네온 사인·울타리(Graphics + 절차 배치). 충돌 오브젝트와 시각 오브젝트 분리.
- **분위기 레이어**: 떠다니는 파티클(먼지/별), 존 주변 광원(글로우), 약한 비네팅. compositor 친화적(알파/스케일 트윈만).
